1. DESCRIPCIÓN DEL PROYECTO
   Sistema local para el control y gestión de ventas de pollos.
   Permite registrar ventas con datos del cliente, cantidades, precios,
   pesos en kilogramos, y control de pagos (total, parcial, pendiente).
   Incluye dashboard estadístico y búsqueda avanzada.

================================================================================

2. TECH STACK
   - Frontend:  HTML5 + CSS3 + Vanilla JavaScript
   - Backend:   Node.js + Express.js
   - Base Datos: SQLite3 (better-sqlite3)
   - Diseño UI: Stitch (Google) + CSS Custom (glassmorphism)
   - Puerto:    http://localhost:3000

================================================================================

3. ARQUITECTURA DE BASE DE DATOS

   Tabla: ventas
   ┌────────────────────┬──────────┬──────────────────────────────────┐
   │ Campo              │ Tipo     │ Restricciones                    │
   ├────────────────────┼──────────┼──────────────────────────────────┤
   │ id                 │ INTEGER  │ PRIMARY KEY AUTOINCREMENT        │
   │ fecha              │ TEXT     │ NOT NULL, DEFAULT datetime local  │
   │ cliente_nombre     │ TEXT     │ NOT NULL, CHECK(length >= 2)     │
   │ descripcion        │ TEXT     │ DEFAULT ''                       │
   │ cantidad_pollos    │ INTEGER  │ NOT NULL, CHECK(> 0)             │
   │ precio_por_kg      │ REAL     │ NOT NULL, CHECK(> 0)             │
   │ peso_total_kg      │ REAL     │ NOT NULL, CHECK(> 0)             │
   │ total_venta        │ REAL     │ NOT NULL, CHECK(>= 0)            │
   │ estado_pago        │ TEXT     │ pendiente | parcial | pagado     │
   │ monto_pagado       │ REAL     │ NOT NULL, DEFAULT 0              │
   │ monto_pendiente    │ REAL     │ NOT NULL, DEFAULT 0              │
   │ created_at         │ TEXT     │ NOT NULL, DEFAULT datetime local  │
   │ updated_at         │ TEXT     │ NOT NULL, DEFAULT datetime local  │
   └────────────────────┴──────────┴──────────────────────────────────┘

   Indices:
   - idx_ventas_cliente (cliente_nombre) — Búsqueda por nombre
   - idx_ventas_estado  (estado_pago)    — Filtro por estado
   - idx_ventas_fecha   (fecha DESC)     — Ordenamiento

   Triggers:
   - trg_ventas_updated_at: Actualiza updated_at al editar

================================================================================

4. API REST — ENDPOINTS

   GET    /api/ventas              → Listar ventas (con ?q=nombre&estado=)
   GET    /api/ventas/:id          → Obtener una venta
   POST   /api/ventas              → Crear nueva venta
   PUT    /api/ventas/:id          → Actualizar venta completa
   PATCH  /api/ventas/:id/pago     → Actualizar estado de pago
   DELETE /api/ventas/:id          → Eliminar venta
   GET    /api/estadisticas        → Estadísticas generales

7. VALIDACIÓN EN TIEMPO REAL

   - Campo vacío → Borde rojo + mensaje de error
   - Número negativo → "Debe ser mayor a 0"
   - Nombre corto → "Mínimo 2 caracteres"
   - Auto-cálculo: Total = Peso (Kg) × Precio/Kg
   - Monto pagado > total → Error "No puede exceder el total"
   - Feedback visual inmediato (colores verdes/rojos)

================================================================================

8. ESTADÍSTICAS DEL DASHBOARD

   KPI 1: Total de pollos vendidos (suma cantidad_pollos)
   KPI 2: Peso total en Kg (suma peso_total_kg)
   KPI 3: Ganancias totales en USD (suma total_venta)
   KPI 4: Total por cobrar en USD (suma monto_pendiente)
   
   Pills: Ventas pagadas | Parciales | Pendientes | Precio promedio/Kg
